## Why Inline Styles Are Necessary

These styles **cannot** be moved to external CSS because they depend on runtime-calculated values:

### Dynamic Positioning
```tsx
// Clip position based on time and zoom level
left: `${clip.startTime * zoom}px`
width: `${clip.duration * zoom}px`

// Playhead position synced with video playback
left: `${currentTime * zoom}px`

// Time ruler markers at calculated intervals
left: `${marker.time * zoom}px`
```

### Dynamic Colors
```tsx
// Track colors from TRACK_COLORS constant
backgroundColor: TRACK_COLORS[track.type]
// e.g., { video: '#3b82f6', audio: '#10b981', image: '#8b5cf6' }
```

### Timeline Width
```tsx
// Total timeline width based on duration and zoom
width: `${timeline.duration * zoom}px`
```

## Why Not Alternative Solutions?

### ❌ Tailwind Classes
```tsx
// Won't work - Tailwind needs static values at build time
className={`left-[${left}px]`} // Not generated by JIT
```

### ❌ CSS Custom Properties
```tsx
// Adds complexity without benefit, still counts as "inline style"
<div style={{ '--left': `${left}px` } as React.CSSProperties}>
```

### ❌ useEffect + DOM Manipulation
```tsx
// Performance overhead, breaks React declarative model
useEffect(() => { ref.current.style.left = `${left}px` }, [left]);
```

### ✅ Inline Styles (Current Solution)
- **Performance**: React optimizes inline style updates
- **Readability**: Clear connection between state and styling
- **Maintainability**: All positioning logic in one place
- **Type Safety**: Full TypeScript support
